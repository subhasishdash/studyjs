Local Storage: Usage and Differences from Session storage

Posted on 27th Sep 2018 at 9:00 PM


Local storage is a storage area within the browser. Though it is little less powerful than indexed DB, but it has a lot of benefits for web developers as well as end-users and they are as follows:

  • No need to use or depend on cookies anymore because cookies are included in every server request that affects website performance.
  • The storage limit of local storage is 5MB, much more than cookies and the information is kept in browser only. In case you need more than 5MB storage, opt for indexed DB.
  • Web storage (local and session) is per domain and protocol. Hence, all the pages from a single Origin can store and access same data and data is never shared among various domains.
  • Local storage is never expired, but Session storage data is kept in memory until the browser tab is closed.
  • It is supported in all modern browser.


Check Browser Support:

Before using it, check browser support: 

var isStorageSupported = function()
 { 
    if(typeof(Storage)! == 'undefined') { 
        return true;
    } else {
        return false;
    }
 }


How to Use ?

It is very easy to use Local storage or Session storage. We have 3 methods for the same:
  • setItem(key, value), sets the value in memory using key.
  • getItem(key), gets the value from memory using the key.
  • removeItem(key), removes a key from memory.


Example:

localstorage.setItem('studyjs.com','I am currently browsing studyjs.com'); //sets the key
localstorage.getItem('studyjs.com'); //Fetches the value by key
localstorage.removeItem('studyjs.com'); //removes the key
