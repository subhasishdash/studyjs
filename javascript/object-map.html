Which one to choose while creating Objects in Javascript?(Object or Map)

Posted on 03rd April 2016 at 9:00 PM


We can use both Object and Map while creating a object in JavaScript . Basically Objects are similar to Maps because both will set keys to values, retrieve those values, delete keys, and detect whether something is stored at a key.
But which one is recommended? To know that you must ask yourself few questions listed below:

  • 1.Do you want to add or remove the key-value pairs frequently?
  • 2.Do you want to use keys as anything other than String?
  • 3.Do all values have the same type, and can be used interchangeably?
  • 4.Do you want to write your own API or use something predefined for creating, updating, deleting and doing basic operations on your object?

If all of the above answers are yes then you need to use Map otherwise use Objects

How to create a Map object in JavaScript?

It is similar to creating a normal object .

var myMap = new Map();

Methods Association

What are the methods associated by window when we create a Object or Map?


When we create a Object we get no default methods but when we create a Map we get 10 default methods to work on it and they are listed below.

  • 1.clear() : Removes all key/value pairs from the Map object.
  • 2.delete(key) : Removes any value associated to the key. Returns true if successfully found and deleted the key ,otherwise returns false
  • 3.entries() : Returns a new Iterator object that contains an array of [key, value] for each element in the Map object in insertion order.
  • 4.forEach(callbackFn[, thisArg]) : Calls callbackFn once for each key-value pair present in the Map object, in insertion order. If a thisArg parameter is provided to forEach, it will be used as the this value for each callback.
  • 5.get(key) : Returns the value associated to the key, or undefined if there is none.
  • 6.has(key) : Returns true if key exists, otherwise returns false.
  • 7.keys() : Lists all keys in a new Iterator object that contains the keys for each element in the Map object in insertion order.
  • 8.set(key, value) : Adds a new entry to the map , key/value can be any data type.
  • 9.size : Returns the number of key/value pairs added to the map.
  • 10.values() : Returns a new Iterator object that contains the values for each element in the Map object in insertion order.

Example:

Creating a map and adding values to it :

var myMap = new Map();
//adding integer as key
myMap.set(1, "integer as key");
//adding float as key and value
myMap.set(9.0, 900.0);
//adding object as key
myMap.set(new Object(), window);
//adding function()
myMap.set(function(){}, "my function");

//see the size of map
console.log(myMap.size); // prints 4 because we have added 4 entries

List all keys

//see all keys
for(var key of myMap.keys()) {
	console.log(key); //prints each key in the loop. 
}

List all values

//see all values
for(var value of myMap.values()) {
	console.log(value); //prints each value in the loop. 
}

Accessing both key/value together

//accessing both key/value together
for (var data of myMap.entries()) {
  console.log(data[0] + " = " + data[1]);//prints each key/value in the loop. Index 0 contains key and 1 contains value.
}

As you have now learned about Map in JavaScript , you could decide whether you want all the above facilities by creating a Map simply or to just use Object.
